24.11.05

UI
主観、誰が見ても変わらない事実

UX
人によって感じ方が変わる

webとアプリの

共通点
どちらもサービスを利用するためのもの
違う点
アプリ：専用アプリケーション
webブラウザ：汎用アプリケーション

webサイトの特徴

URLと不可分に結びついているので「戻る」「進む」の制御が派生

メリット
・マルチデバイスの展開のしやすさ
・OSの違いによる環境の差異が小さい
・複数のブラウザを同時に使用できる
・サービスごとのインストールが不要
デメリット
・操作上のインタラクションが乏しい　例）Excel
・ジェスチャーへの対応が乏しい
・操作スピードが遅い

アプリの特徴

ハードウェア的に異なるため、インターフェースの差異が大きい

メリット
・動作が速く、操作が軽快
・タッチパネルでは多様なジェスチャーに対応できる
・データ通信量が（同一条件のwebサイトと比較して）少量で済む
・中断や再開が容易
デメリット
・サービスごとに都度インストールが必要
・OSごとに異なるプログラミング言語で開発する必要がある
・最適なインターフェースがデバイスやOSごとに異なる

テキスト・画像・動画

テキスト
データの基本にして、最強のデータ形式
デバイス、解像度問わず、OSによる最適な表示が担保されている
画像
テキストと並んで重要なデータ形式
どのような場合であっても環境によらず全く同一に見えることが特徴
ベクターデータ（SVG、PDF など）は解像度の影響を受けない
動画
画面が大きいデバイスほどよく視聴される傾向が強い
ほとんどの動画は「16:9」のアスペクト比で固定されているので自由度に制限がある

パフォーマンス

表示と操作に関わる「速さ（スピード）」のこと
表示の速度が遅くなるほど、ユーザーが必要なコンテンツを見つけられる可能性が下がる







24.11.12

インタラクションコスト

頭への負担：最適度以上脳を働かせる　エネルギー消費
→読む、理解する、覚える

体への負担：最適度以上体を動かせる
→クリック/タップする、スクロールする、スワイプする

ユーザビリティの良し悪しを測る指標
インタラクションコスト↓、デザイン↑


一貫性

サービス内にはあるルールに沿ってデザインされていること

メリット

  • 予測できる
  • 機能の追加や改善が簡単になる
  • 信頼性が生まれる

↓

頭への負担が減少

シンプルさ

単純にするではなく、「明快」にすること

明快：わかりやすくて、すっきりした気分になる（p79）

難点
要素が少ないが、目的をちゃんと理解でき、正しく操作できる

注意点
初心者→シンプル、プロ→複雑

共通概念

世の中に、広い範囲で使われているルールのこと
→新たな学習コストを減らす（p82、83）

メリット
脳の負担を低減する
サービス本来の目的にユーザーが集中できるようにする
例）ユニバーサルデザイン（p83、84、85）






24.12.03

ナビゲーション
ユーザーが機能や情報を見つけるのを手助けして、望ましい行動をとってもらうためのインターフェースの集合体

インタラクション
ユーザーの動きに対する動きや変化を伴ったフィードバック

ナビゲーション

ナビゲーションの配置とパターン
ナビゲーションを配置する場所は上下左右のいずれかが基本
ヘッダー、フッターパンくずリスト、グローバルナビゲーション、ローカルナビゲーション、関連ナビゲーション、オーバーレイ、インレイなどがよく登場する

コンテンツの量
コンテンツの量が増えれば、設置するナビゲーションの巣類も増える

画面エリアの自由度
画面エリアの「大きさ」と「固定が可能なのか」によって最適なナビゲーションが異なる

入力手段の違い
細かいパーツで構成されているナビゲーションの操作は、マウスでは問題なくてもタッチ操作やリモコン操作においては不適切になり、よりおおまかで手数の少ないものが必要になる

インタラクション

インタラクションはその目的とするところから大きく、理解のためのインタラクションと演出のためのインタラクションの大きく2種類に分類される

理解のためのインタラクション
より明確な理解を促すための動き
使いやすさと分かりやすさに直結するインタラクション

演出のためのインタラクション
素敵な体験を介してサービスの見地区を向上させる動き
ブランドの認知を訴求するために効果的なインタラクション

マイクロインタラクション
製品やサービスにおける最小単位のインタラクション







24.01.21

デザインガイドライン

デザインガイドラインの目的

色・文字・レイアウト・表現方法といった様々なデザイン要素について、ルールを厳密に定義したドキュメント

Apple：ヒューマンインターフェースガイドライン（Human Interface Guidelines）
IOS7以降のAppleのフラットデザインに基づいたデザイン指針をまとめたもの
デバイス固有の特徴を活かすためのコンセプトとして存在している

Google：マテリアルデザイン（Material Design）
Googleによるミニマルデザインのガイドライン
インタラクションによる説明が多用されているため、視覚的に理解しやすい構造になっている

Microsoft：マイクロソフトデザイン（Microsoft Design）
Microsoftが公式で提唱しているデザインガイドライン
Natural on each device（どのデバイスでも自然に）、Intuitive and powerful（直観的で力強く）、Engaging and immersive（魅力的で没入できる）というテーマのもと、ARやウェアラブルを含めた世界観を表現するデザイン体系を目指している

デザインガイドラインの作成

実際のプロジェクトにあたって、デザイナーがサービスデザインに携わる重要な役割の一つが、単にデザインをするだけでなくデザインを通してインターフェースデザインのルールを作り、継続的に運用しやすいサービスに仕上げること
